Implement URL patch mapping rules to scripts Example

The webserver will load _mapping_rules_ from the current path. If matches on a url path based using regular expressions succeed then it runs the associated script in the mapping rule.

If you place a _mapping_rules_ file in the root website, then it will map to any path. If a _mapping_rules_ file exists in the sub-directory of a website, it will take precedence over a root defined _mapping_rules_

If no _mapping_rules_ rule is defined, then the web server will try to execute index.py, or index.ty if the url path matches a website's directory; working just like the root path default of a website.

To escape the comma char, place 2 commas next to each other.

# comment
regexp, <regexp matching relative path>, <web server template/python file in current directory to run>
regexp-opt, <regexp matching relative path>, <web server template/python file in current directory to run>, <python regexp options>
pymatch, <python code matching on variables: basepath, querystring>, <web server template/python file in current directory to run>
